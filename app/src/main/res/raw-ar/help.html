Backup modes: An app can be backed up with only its apk, only its data or both. An app not currently installed can only be restored if the apk is present.

Delete backup: Deletes the backup files for the chosen app.

Uninstall: Somewhat more aggressive than a normal uninstall. In addition to doing a normal uninstall via android commands, uninstalling from OAndBackupX deletes files the app might have left over in /data/app-lib/.

Enable / disable: Uses the android script `pm` to enable or disable an app. Disabling an app removes it from the normal user interface without uninstalling. This can be used for enabling or disabling an app for muliple users at a time (if the device has multiple users enabled). Users are identified with an id. 0 is the first user (owner).

مستخدمين متعددين: لا يزال المستخدم المتعدد تجريبيا إلى حد ما ولكن يجب أن يعمل. تمكين وتعطيل يعمل فقط بعد التثبيت الأولي (وليس بالضرورة من OAndBackupx) أو استعادة التطبيق.

تطبيقات بدون نظام : النسخ الاحتياطي واستعادة التطبيقات التي لا تحتوي على نظام مثبتة لأن وحدات Magisk غير مدعومة وقد تؤدي تجربتها إلى فشل النظام وإجبارك على اعادة التهيئة وتثبيتها من جديد.

التنقل: يحتوي شريط التنقل على "إعدادات" و"الرئيسية" و"النسخ الاحتياطية المتتابعة" و"استعادة الدفعة" و"جدولة النسخ الاحتياطية" (بهذا الترتيب). سيتم تقديم عدد التطبيقات التي تمت ترقيتها بعد آخر نسخة احتياطية لها كإشعارات على زر "الصفحة الرئيسية".

التطبيقات التي تستخدم Keystore: يحتوي Android على مساحة تخزين مشفرة للبيانات الحساسة. لا يمكن استعادة التطبيقات التي تستخدم هذا لتخزين بيانات تسجيل الدخول أو بعض مفاتيح وظيفتها بنجاح. مثالين من هذه التطبيقات هي Signal و Element(Riot).
