Modes de sauvegarde : Il est possible de ne sauvegarder que l'APK d'une app, que ses données, ou bien les deux. Une app qui n'est pas installée ne peut seulement être restaurée si l'APK a été sauvegardée. Trois types spéciaux de données sont pris en charge en plus des données principales : les données protégées par l'appareil, les données externes et les données obb.

Supprimer la sauvegarde : Supprime une sauvegarde spécifique de l'application sélectionnée. "Supprimer les sauvegardes" supprime toutes les sauvegardes de l’application sélectionnée.

Désinstaller : Un peu plus aggressif qu'une désinstallation normale. En plus d'une désinstallation normale via les commandes Android, OAndBackupX supprime les fichiers que l'appli aurait pu laisser dans /data/app-lib/.

Activer / désactiver : Utilise le script Android `pm` pour activer ou désactiver une app. Désactiver une app la retire de l'interface utilisateur normale sans la désinstaller. Il est aussi possible d'activer ou désactiver une app pour plusieurs utilisateurices à la fois (si l'appareil est configuré pour). Les utilisateur·ices sont identifié⋅e·s par un identifiant. Le premier est le numéro 0 (propriétaire).

Utilisateurs multiples  : Le multi-utilisateur est encore quelque peu expérimental mais devrait fonctionner. L'activation et la désactivation ne fonctionnent qu'après une installation initiale (pas nécessairement à partir d'OAndBackupx) ou une restauration de l'application.

Applications sans système : La sauvegarde et la restauration des applications sans système installées en tant que modules Magisk ne sont pas prises en charge et leur tentative pourrait entraîner une défaillance du système et vous obliger à le formater et à le flasher à nouveau.

Navigation : La barre de navigation a « Paramètres », « Accueil », « Sauvegarde par lots », « Restauration de lots » et « Sauvegardes d’horaire » (dans cet ordre). Le nombre d’applications mises à niveau après leur dernière sauvegarde sera présenté sous forme de notification sur le bouton « Accueil ».

Applications utilisant Keystore : Android dispose d'un stockage chiffré pour les données sensibles. Les applications qui l'utilisent pour stocker des données de connexion ou certaines clés pour sa fonction ne peuvent pas être restaurées avec succès. Signal et Element (Riot) sont deux exemples de telles applications.
