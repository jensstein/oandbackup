Modos de Backup: Um app pode ser copiado apenas com seu APK, apenas com seus dados ou ambos. Um app não instalado atualmente só pode ser restaurado se o APK estiver presente. Três tipos especiais de dados são suportados além dos dados principais: dados protegidos do dispositovo, dados externos e dados OBB.

Excluir backup: Apaga os arquivos de backup para uma instância de backup específica. Enquanto "excluir backups" apaga todos os backups do app escolhido.

Desinstalar: Um pouco mais agressivo que uma desinstalação normal. Além de fazer uma desinstalação via comandos Android, desinstalar do OAndBackupX exclui arquivos que o app pode ter deixado em /data/app-lib/.

Ativar / Desativar: Usa o script `pm` do Android para ativar ou desativar um app. Desativiar um app remove-o da interface normal do usuário sem desinstalá-lo. Isto pode ser usado para ativar ou desativar um app para vários usuários ao mesmo tempo (se o dispositivo tiver vários usuários). Os usuários são identificados com um ID, por exemplo, 0 é o primeiro usuário (proprietário), 10 é normalmente o perfil de trabalho.

Múltiplos usuários: Multiusuário ainda é um pouco experimental, mas deve funcionar. Ativar e desativar só funciona após uma instalação inicial (não necessariamente do OAndBackupX) ou restauração do app.

Apps Sem Sistema: Fazer backup e restaurar apps sem sistema instalados como módulos Magisk não é suportados e tentar isso pode levar à falha do sistema e forçá-lo a formatar e atualizar novamente.

Navegação: A barra de navegação possui "Configurações", "Início", "Backup em lote", "Restauração em lote" e "Agendar backups" (nessa ordem). O número de apps atualizados após seu último backup será apresentado como uma notificação no botão "Início".

Apps usando Keystore: O Android tem um armazenamento criptografado para dados confidenciais. Os aplicativos que usam isso para armazenar dados de login ou algumas chaves para sua função não podem ser restaurados com sucesso. Dois exemplos de tais aplicativos são Signal e Element (Riot).
