Modos de copia de seguridad: se puede hacer una copia de seguridad de una aplicación solo con su apk, solo con sus datos o con ambos. Una aplicación que no está instalada actualmente solo se puede restaurar si el apk está presente. Se admiten tres tipos especiales de datos adicionales a los datos principales: datos protegidos del dispositivo, datos externos y datos obb.

Eliminar copia de seguridad: elimina los archivos de copia de seguridad de una instancia de copia de seguridad específica. Mientras que "eliminar copias de seguridad", elimine todas las copias de seguridad de la aplicación elegida.

Desinstalación: Algo más agresiva que una desinstalación normal. Además de realizar una desinstalación normal a través de los comandos de Android, la desinstalación desde OAndBackupX elimina los archivos que la aplicación pueda haber dejado en /data/app-lib/.

Habilitar / deshabilitar: Utiliza el script de Android " pm " para habilitar o deshabilitar una aplicación. Desactivar una aplicación la elimina de la interfaz de usuario normal sin desinstalarla. Se puede utilizar para habilitar o deshabilitar una aplicación para varios usuarios a la vez (si el dispositivo tiene varios usuarios). Los usuarios se identifican con un ID, por ejemplo, 0 es el primer usuario (propietario) 10 suele ser el perfil de trabajo.

Usuarios Múltiples: El modo multiusuario es todavía algo experimental pero debería funcionar. Habilitarlo y deshabilitarlo sólo funciona después de una instalación inicial (no necesariamente desde OAndBackupx) o restaurando la aplicación.

Aplicaciones System-less: el respaldo y la restauración de las aplicaciones System-less como los módulos de Magisk , no es compatible y el intento podría conducir a un fallo del sistema y obligar a formatear y flashear de nuevo.

Navegación: la barra de navegación tiene "Configuración", "Inicio", "Copia de seguridad por lotes", "Restauración por lotes" y "Programar copias de seguridad" (en ese orden). El número de aplicaciones actualizadas después de su última copia de seguridad se presentará como una notificación en el botón "Inicio".

Aplicaciones que usan Keystore: Android tiene un almacenamiento encriptado para datos confidenciales. Las aplicaciones que hacen uso de esto para almacenar datos de inicio de sesión o algunas claves para su función no se pueden restaurar con éxito. Dos ejemplos de tales aplicaciones son Signal y Element (Riot).
