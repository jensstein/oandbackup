примітки щодо використання

режими резервного копіювання: додаток може бути зарезервовано тільки з його apk, даними, або і тим і іншим. програму, що не встановлено, може бути відновлено тільки при наявності apk.

контекстне меню: натисніть і утримуйте елемент у списку програм, щоб отримати контекстне меню.

видалення резервної копії: видаляє файли резервного копіювання для вибраного додатка.

деінсталяція: дещо агресивніша, ніж звичайна деінсталяція. на додаток до звичайної деінсталяції за допомогою команд андроїд, деінсталяція з oandbackup видаляє файли, які програма, можливо, залишила в /data/app-lib/. також працює для системних додатків.

включити/відключити: використовує андроїд-скрипт `pm`, щоб увімкнути або вимкнути додаток. відключення програми видаляє його зі звичайного інтерфейсу користувача без деінсталяції. використовується для ввімкнення або вимкнення програми для кількох користувачів одночасно (якщо на пристрої увімкнено кілька користувачів). користувачі розпізнаються по ідентифікатору. 0 - перший користувач (власник).

кілька користувачів: багатокористувацький режим все ще дещо експериментальний, але повинен працювати. при відновленні в багатокористувацькому сценарії після команди `pm install -r $ apk` додаток буде вимкнено для кожного користувача, чий додаток вказано в /data/system/user/$user/package-restrictions.xml (якщо додаток не вказано як увімкнено="1").
це може створити проблеми для користувачів, які встановлюють один і той самий додаток згодом, але це необхідно для того, щоб програма не могла встановлюватися одночасно для всіх користувачів. контекстне меню дає можливість вмикати або вимикати програми, які можна використовувати, якщо інші користувачі не можуть використовувати певну програму через вимкнення під час відновлення.
увімкнення та вимкнення функціонує лише після початкової установки (не обов'язково з oandbackup) або відновлення програми.
