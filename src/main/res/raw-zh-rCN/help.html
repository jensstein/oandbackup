使用说明

备份模式:要备份一个应用只能备份它的安装包或/与数据.目前尚未安装的应用只有应用存在时才能恢复.

上下文菜单:长按应用列表中的某个项目即可获取上下文菜单

删除备份:删除所选应用的备份文件

卸载:比正常卸载更强. 除了通过 Android 命令进行正常卸载, 使用 oandbackup 提供的卸载功能还可以删除应用程序在 /data/app-lib/ 目录里所残留的文件. 这也适用于系统应用.

启用/停用:使用 Android 脚本`pm`启用或禁用应用. 禁用应用会将其从正常用户界面中移除而无需卸载. 如果设备已启用多个用户, 这也可以一次性为多个用户启用或禁用应用. 被标识为 id. 0 的用户是第一个用户(所有者).

多用户:多用户功能虽然处于实验性阶段, 但应该能正常工作. 当在多用户场景中恢复时, `pm install -r $apk`被调用,并且随后在 /data/system/user/$user/package-restrictions.xml 中被列出的应用对于每一个已安装它的用户都是处于禁用(除非应用程序被列为enabled ="1")
这可能会为稍后安装相同应用的用户带来问题, 但必须防止相同的应用同时被安装到所有用户上. 如果其他用户由于在恢复时禁用而无法使用特定应用, 你可以使用上下文菜单选项中提供的"启用或禁用该应用"这一个功能来解决此问题.
启用和禁用仅在初始安装或恢复应用之后有效, 此操作不要求必须在 oandbackup 里执行.
