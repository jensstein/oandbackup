примечания по использованию

режимы резервирования: приложение может быть зарезервировано только с его apk, данными, либо и тем, и другим. приложение, которое не установлено, может быть восстановлено только при наличии apk.

контекстное меню: сделайте долгое нажатие на пункте в списке приложений, для доступа к контекстному меню.

удалить р.к.: удаляет файлы р.к. для выбранного приложения.

деинсталляция: в некотором роде более агрессивна, чем обычная деинсталляция. в дополнение к обычной деинсталляции при помощи команд android, деинсталляция из oandbackup удаляет файлы, которые приложение могло оставить в /data/app-lib/. она так же работает для системных приложений.

включить / отключить: использует скрипт android `pm` для включения или отключения приложения. отключение приложения удаляет его из обычного пользовательского интерфейса без деинсталляции. это может быть использовано для включения или отключения приложения для нескольких пользователей одновременно (если на устройстве включено несколько пользователей). пользователи определяются по идентификатору. 0 это первый пользователь (владелец).

несколько пользователей: многопользовательский режим это все еще нечто экспериментальное, но должно работать. при восстановлении в многопользовательском режиме вызывается `pm install -r $apk` и впоследствии приложение отключено для всех пользователей, у которых приложение присутствует в /data/system/user/$user/package-restrictions.xml (если только приложение не отмечено как включенное ="1").
это может создать проблемы для пользователей, устанавливающих одинаковое приложение позже, но необходимо предотвратить установку приложения для всех пользователей одновременно. в контекстном меню есть опция для включения или отключения приложений, что может быть использовано, если другие пользователи не могут использовать определенные приложения из-за отключения при восстановлении.
включение и отключение работает только после начальной установки (не обязательно из oandbackup) или восстановления приложения.
